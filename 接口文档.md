## 认证相关

### 1. 用户登录

**POST** `/api/v1/login/access-token`

获取访问令牌（OAuth2 兼容）

**请求参数**:
- `username` (form-data, string, 必填): 用户邮箱
- `password` (form-data, string, 必填): 用户密码

**响应**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

**认证**: 无需登录

---

### 2. 测试令牌

**POST** `/api/v1/login/test-token`

测试访问令牌是否有效

**请求头**:
- `Authorization: Bearer {token}`

**响应**: 返回当前用户信息

**认证**: 需要登录

---

### 3. 密码恢复

**POST** `/api/v1/password-recovery/{email}`

发送密码重置邮件

**路径参数**:
- `email` (string, 必填): 用户邮箱

**响应**:
```json
{
  "message": "Password recovery email sent"
}
```

**认证**: 无需登录

---

### 4. 重置密码

**POST** `/api/v1/reset-password/`

使用重置令牌重置密码

**请求体**:
```json
{
  "token": "reset_token_here",
  "new_password": "new_password_here"
}
```

**响应**:
```json
{
  "message": "Password updated successfully"
}
```

**认证**: 无需登录

## 用户管理

### 1. 获取用户列表

**GET** `/api/v1/users/`

**响应**:

**认证**: 需要管理员权限

---

**POST** `/api/v1/users/`

## 通用说明

### 认证方式

大部分接口需要认证，使用 Bearer Token 方式：

```
Authorization: Bearer {access_token}
```

获取 token 的方式：
1. 通过 `/api/v1/login/access-token` 接口登录获取
2. 通过 `/api/v1/users/signup` 注册后登录

### 响应格式

所有接口统一返回格式：

**成功响应**:
- 200: 成功
- 201: 创建成功

**错误响应**:
- 400: 请求参数错误
- 401: 未认证
- 403: 权限不足
- 404: 资源不存在
- 500: 服务器错误

错误响应格式：
```json
{
  "detail": "错误描述信息"
}
```

### 分页参数

列表接口支持分页：
- `skip`: 跳过记录数（默认: 0）
- `limit`: 返回记录数（默认: 100，最大: 1000）

### 权限说明

- **无需登录**: 公开接口，任何人都可以访问
- **需要登录**: 需要有效的访问令牌
- **需要管理员权限**: 需要登录且用户具有超级管理员权限
